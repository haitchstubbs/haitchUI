# Typography

A set of shadcn-aligned typography primitives designed to be used both directly in React and automatically through MDX rendering.

## Installation


Import to next.config.ts

```ts
import createMDX from "@next/mdx"
import remarkGfm from "remark-gfm"

/** @type {import("next").NextConfig} */
const nextConfig = {
  pageExtensions: ["js", "jsx", "ts", "tsx", "mdx"],
}

const withMDX = createMDX({
  options: {
    remarkPlugins: [remarkGfm],
  },
})

export default withMDX(nextConfig)
```

## Accessibility and semantics

All typography elements render semantic HTML elements:

* Headings render `h1`–`h6`
* Paragraphs render `p`
* Lists render `ul`, `ol`, and `li`
* Tables render `table`, `tr`, `th`, and `td`

All elements support native HTML attributes such as `id`, `aria-*`, and alignment attributes.

---

## Recommended usage

* Use MDX for documentation and long-form content.
* Use typography primitives directly in TSX for UI text.
* Keep typography primitives focused on hierarchy and spacing, not layout.

---

## Summary

This typography system provides:

* Consistent visual hierarchy
* Semantic HTML output
* Automatic MDX rendering
* Shadcn-compatible styling
* Minimal API surface

It is intentionally boring — and that’s the point.
